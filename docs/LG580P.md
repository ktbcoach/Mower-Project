# LG580P GNSS logger (fork of the Watson logger)

Parallel logger for the **SparkFun LG580P GNSS Flex pHAT** (Quectel LG580P —
dual-antenna RTK GNSS with heading). Lives alongside `watson_dms`; the Watson
logger is preserved and untouched.

## Hardware

| | |
|---|---|
| Receiver | SparkFun LG580P Flex pHAT (dual-antenna RTK + heading, up to 20 Hz) |
| Interface | Pi **primary UART**, GPIO14 (TX) / GPIO15 (RX) → `/dev/serial0` |
| Baud | **460800** (confirmed default; `detect` sweeps 9600–921600) |
| Output | NMEA 0183 (GGA, RMC, VTG, GSA, GSV…) + Quectel **PQTM** (config + `PQTMTAR` heading) |
| RTK status | GGA fix-quality field: 4 = RTK fixed, 5 = RTK float |
| Switch/LEDs | Multi-IO HAT dry-contact + LEDs (coexists — different pins) |

## Setup on the Pi

```bash
sudo bash scripts/setup_pi_lg580p.sh      # enable primary UART (PL011), disable console
sudo reboot
# (watson_dms + lg580p install together)
cd ~/gps-collector && source .venv/bin/activate
pip install -e ".[pi]"
```

## Bring-up

```bash
python -m lg580p detect                    # find the baud rate
python -m lg580p capture --baud 460800 --seconds 5   # see raw sentences + a type summary
```

The `capture` summary lists which sentence types appear (and flags bad
checksums) — use it to confirm the receiver is emitting GGA/RMC/VTG and the
`PQTMTAR` heading message.

## Logging

```bash
# Continuous (writes logs/lg580p-<timestamp>.csv, live status line):
python -m lg580p collect --gpx logs/track.gpx

# Switch-gated field service (Multi-IO HAT dry-contact ch1 gates logging):
python -m lg580p collect --switch

# Auto-start at boot:
sudo bash scripts/install_lg580p_service.sh
journalctl -u lg580p -f
```

LED status (HAT): **GPS LED** off=no fix · blink=fix (GPS/DGPS/RTK-float) ·
solid=**RTK fixed**. **Logging LED** off=idle · blink=logging.

> Only run one service at a time — `lg580p` and `watson-dms` both grab a serial
> port and the HAT. `sudo systemctl disable --now watson-dms` before enabling
> this one.

## Dual-antenna heading setup

Heading is computed from the vector between the two antennas. Preloading the
**fixed baseline distance** (their exact center-to-center separation) constrains
RTK ambiguity resolution so heading resolves faster and more reliably. Stop the
logger/service first (the port must be free), then:

```bash
# measure the antenna separation precisely; e.g. 1.000 m:
python -m lg580p config set-baseline 1.000      # sends PQTMCFGBLD + PQTMSAVEPAR
# then power-cycle the module.

python -m lg580p config get-baseline            # read it back
python -m lg580p config send "PQTMCFGRCVRMODE,W,1" --save   # raw command (rover mode)
```

`config` builds the checksum for you and prints the receiver's `$PQTM…`
acknowledgement. Baseline range 0.000–5.000 m; `0` = auto-calculate.

> If heading still won't solve after setting the baseline, the issue is upstream:
> both antennas must be connected with a clear sky view and a real physical
> separation — a preset baseline can't fix a missing/blocked second antenna.

## Notes

- **Heading** comes from `PQTMTAR` field 7 (0–360°); the receiver's `$GNTHS` can
  stay void even while `PQTMTAR` reports heading, so `PQTMTAR` is the primary
  source. `PQTMTAR` order (confirmed): `MsgVer,UTC,Quality,Res,Length,Pitch,
  Roll,Heading,AccPitch,AccRoll,AccHeading,UsedSV`. Quality 4 = RTK fixed,
  5 = RTK float. A 2-antenna system leaves **Roll** empty (heading + pitch only).
- `GnssReading` reuses `latitude_deg` / `longitude_deg` / `altitude_m` /
  `has_gps_fix`, so a GPX writer can log it directly.
